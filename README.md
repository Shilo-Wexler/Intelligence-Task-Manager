# Intelligence-Task-Manager

## Project Goal
This project simulates a system that is supposed to manage agents and tasks for a unit called ShadowNet

The system's emphasis is on risk levels per task and task status management

as well as on managing agents and agent ranks and assigning tasks to them.

## SQL Table Structure

### Agent Storage Table:

| Field | Description | 
| :---- | :---------- |
| id    | A unique identifier is automatically generated by the database. |
| name  | Agent name. |
| specialty | The agent's area of ​​expertise. |
| is_active | Identifier, True when the agent is active False otherwise. | 
| completed_missions | Counting tasks completed by the agent. |
| failed_missions | Counting the number of tasks that the agent failed. |
| agent_rank | Contains  agent rank from the options {Junior / Senior / Commander} only. |

### Task Storage Table:

| Field | Description | 
| :---- | :---------- |
| id    | A unique identifier is automatically generated by the database. |
| title  | Mission title. |
| description | Detailed description of the task. |
| location | Mission location. | 
| difficulty | Difficulty level of the task from 1 to 10 (10 being the most difficult). |
| importance | Task importance from 1 to 10 (with 10 being the most difficult). |
| status | Task status when NEW is the default. |
| risk_level | Risk level including calculation of importance and difficulty (internal system calculation.) |
| assigned_agent_id | When assigning a task to an agent, this field will contain the agent ID. |

## Detail Of The Methods That Can Be Executed In The Database

### ConnectionDB class:

The class that creates and manages the connection to the database (OOP implementation).

| Method              |  Role | 
| :-----------------  | :---- |
| `get_connection()`  |  Returns a working connection to the database every time (the scope of the project is small). |
| `create_database()` | Creates the database (checks if it doesn't exist before creating to avoid overwriting). |
| `create_tables()`   | Creates the storage tables (checks that they do not exist to avoid data loss). |


### MissionDB class 

The class that manages the operations for updating, creating, and receiving data related to tasks.

| Method              |  Role | 
| :-----------------  | :---- |
| `create_mission(data)`  | Creates a new task |
| `get_all_missions()` | Returns all tasks |
| `get_mission_by_id(id)`   | Returns one task by ID, or None |
| `assign_mission(m_id, a_id)`   | Assigning a task to an agent |
| `update_mission_status(id, status)`   | Used for any status change |
| `get_open_missions_by_agent(id)`   | Returns agent ASSIGNED/IN_PROGRESS tasks |
| `count_all_missions()`   | Total tasks |
| `count_by_status(status)`   | Total tasks count by specific status |
| `count_open_missions()`   | Total tasks Open task counter |
| `count_critical_missions()`   | Total CRITICAL tasks |
| `get_top_agent()`   | The agent with the highest completed_missions |


### AgentDB Class

The class that manages the operations for updating, creating, and receiving data Related to agents. 

| Method              |  Role | 
| :-----------------  | :---- |
| `create_agent(data)`  | Add a new agent. |
| `get_all_agents()` | Returning a list of all agents. |
| `get_agent_by_id(id)`   | Returns one agent by ID, or None. |
| `update_agent(id, data)`   | Updating agent details. |
| `deactivate_agent(id)`   | Sets agent inactive status |
| `increment_completed(id)`   | Updates the number of tasks completed. |
| `increment_failed(id)`   | Updates the number of failed tasks. |
| `get_agent_performance(id)`   | Returns a dictionary with the keys (completed, failed, total, success_rate). |
| `count_active_agents()`   | Returns the number of active agents. |


## System Rules (business logic)


| [i] | Rule: | 
| :--- | :---------- |
| I    |   Rank must be {Junior / Senior / Commander}. |
| II   |   difficulty and importance must be between 1 and 10 — otherwise an error. |
| III  |   risk_level is calculated automatically when a task is created—the user does not submit it. | 
| VI   | An agent with is_active=False cannot accept tasks. | 
| V    | An agent cannot have more than 3 open tasks (ASSIGNED / IN_PROGRESS) at the same time. |
| IV   | If risk_level=CRITICAL — only an agent with the rank of Commander can accept the task. |
| IIV  | Only a task with a status of NEW can be assigned. After assignment: status=ASSIGNED. |
| IIIV | Only a task with the status ASSIGNED can be started. After: status=IN_PROGRESS. |
| XI   | You can only complete a task in IN_PROGRESS status and change it to failed or completed status. |
| X    | Only a task with a status of NEW or ASSIGNED can be canceled — otherwise an error. | 



## How To Run

1 -> Run in the terminal To download the project:

```
git clone https://github.com/Shilo-Wexler/Intelligence-Task-Manager.git;
```

2 -> Run in the terminal To create a safe environment:

```
cd Intelligence-Task-Manager.git; python3 -m venv venv; source venv/bin/activate;
```

3 -> Run in terminal to install required libraries

```
pip install -r requirements.txt
```

4 -> Run in the computer terminal to start the container for the database:

```
docker run -d --name intelligence-mysql -e MYSQL_ROOT_PASSWORD=1234 \
  -e MYSQL_DATABASE=Intelligence_db -p 3306:3306 mysql:8.0
```
